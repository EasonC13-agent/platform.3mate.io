Overview

3mate Platform is a generic micropayment layer for API services, built on Sui. It uses a payment channel (Tunnel) pattern where users deposit funds on-chain and consume them off-chain via signed receipts, with on-chain settlement.

State Receipt Signing

Each API call updates the cumulative usage amount. The backend signs a state receipt using the API key's Ed25519 keypair:

message = tunnel_id_bytes || bcs::to_bytes(cumulative_amount) || bcs::to_bytes(nonce)
signature = ed25519_sign(message, private_key)

The operator can submit this signature on-chain to claim funds up to cumulative_amount. The contract verifies the signature against any authorized key in the Tunnel.

Quick Start

1. Connect Wallet

Visit platform.3mate.io and connect your Sui wallet. Make sure you're on Testnet.

2. Mint Test USDC

Click "Mint TEST_USDC" on the dashboard to get free test tokens for development.

3. Open a Tunnel

Deposit USDC to create a payment Tunnel. This locks your funds in an on-chain escrow contract.

4. Create an API Key

Generate an API key from the dashboard. The key is shown once. Copy it immediately.

    "messages": [{"role": "user", "content": "Hello!"}]
  }'

Integration Guide

Want to add Tunnel payments to your own service? Here's how.

For API Consumers (Apps)

If you're building an app that consumes LLM APIs (like LuLuAI), integration is simple:

  1. Get an API key from platform.3mate.io
  2. Replace your Anthropic base URL with https://platform.3mate.io
  3. Use your sk-3mate-... key in the x-api-key header
Python
import anthropic

client = anthropic.Anthropic(
    api_key="sk-3mate-...",
    base_url="https://platform.3mate.io"
)

message = client.messages.create(
    model="claude-haiku-4-5",
    max_tokens=1024,
    messages=[{"role": "user", "content": "Hello!"}]
)
TypeScript
import Anthropic from '@anthropic-ai/sdk';

const client = new Anthropic({
  apiKey: 'sk-3mate-...',
  baseURL: 'https://platform.3mate.io'
});

const message = await client.messages.create({
  model: 'claude-haiku-4-5',
  max_tokens: 1024,
  messages: [{ role: 'user', content: 'Hello!' }]
});

For Service Providers

To offer your own API service with Tunnel payments:

  1. Deploy a CreatorConfig on Sui with your operator key
  2. Run a backend that verifies API keys against on-chain Tunnel state
  3. Periodically call claim() to settle accumulated usage

See the source code for a complete reference implementation.

Security Model

Trust Assumptions

  • Payer trusts operator to provide the service and claim fairly
  • Operator trusts on-chain state for deposit verification
  • Neither party can steal: funds are locked in contract, claims require valid signatures, payer can always close

Key Management

  • API keys are Ed25519 keypairs. The private key IS the API key
  • Public keys are registered on-chain via add_authorized_key()
  • Backend stores only public key + hint (first/last chars). Full key shown once at creation
  • Compromised key? Remove it on-chain with remove_authorized_key() and generate a new one

Grace Period

When a payer initiates close, there's a grace period (default: 60 minutes) for the operator to submit final claims. After the grace period, the payer can finalize and withdraw remaining funds.

Cooperative Close

The operator can also close the Tunnel immediately with a final signed receipt, returning the remaining balance to the payer in one transaction.
